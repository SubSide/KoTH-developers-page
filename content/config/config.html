Global

use-cache

This enables the caching in the plugin

Caching is used for making the KoTH continue between reloads

Also saves the maprotation index at reloads and restarts

debug

Enables debugging in the plugin

Currently only used to show debug info about schedules when they're loaded.

multiplekothsatonce

If disabled only 1 KoTH can run at any time

This can be overwritten with the KothStartEvent

schedule-show-current-day-only

When a player does /koth schedule, it will only display the entries of that day.

schedule-timezone

The timezone of where you live at, if the timezone doesn't work, try one with the same time offsets.

Default: "Europe/Amsterdam" (pssst, that's where I live ;) )

You can grab your timezone here: Wikipedia

minuteoffset

This is to adjust the time you're seeing in /koth schedule

Together with startweekminuteoffset should be used to adjust the time when schedule-timezone isn't working

startweekminuteoffset

This is to create an offset for when the start of the week begins

Together with minuteoffset should be used to adjust the time when schedule-timezone isn't working

pre-broadcast

This defines how many seconds before a schedule starts a broadcast should appear

nocap-broadcast-interval

This defines the interval of seconds between each broadcast that nobody caps

helpcommand

The help menu that players see

Loot

default

the default loot that the plugin should use, for when there is none set for the koth.

randomize

Randomize loot will make it so it will randomly generate the loot for you, If you don't want this (You want all the items to be given in the chest) You should put this to false.

default-amount

This is the default amount of loot that will be randomly selected from the loot chest.

randomize-stacksize

The randomize-stacksize chooses a random size of the stack you given. Say you put in 32 cobblestone, it can choose a random amount between 1 and 32.

can-use-same-items

can-use-same-items is for if it can use the same item multiple times.

remove-after

remove-lootchest-after will remove the loot chest after this amount of seconds. Do note that if the server gets reloaded/restarted that this (might) not be the case!

0 to disable this

drop-on-removal

drop-loot-on-removal makes it so when the loot chest gets removed, it drops the loot instead of it just disappearing. default is false

give-instantly

give-instantly should be put to true if you want the winner to immediately get all items instead of in a chest. This can be very useful for crate keys and such. Putting this to true means that no chest will spawn!

commands

WARNING! KoTH is executing commands with NO limitations.
For safety, at least make sure that only OP's can access this command

enabled: false

Putting this to true allows you to link commands to KoTH

changeingame: false

If you want to change the commands from ingame, set this to true. Make sure only people you can absolutely trust can edit them though!

needop: true

Setting this to false makes it so you only need the permission to edit the commands.

hooks

vanishnopacket

Having this enabled makes it so that people that are in vanish are not able to capture the KoTH

factions

If you want people to be able to capture a KoTH as a faction, enable this.

This is for both Factions AND FactionsUUID!

kingdoms

Same as the Factions hook, enable this if you want people to be able to capture a KoTH as a kingdom.

featherboard

enabled: false

Set this to true if you want to use the Featherboard hook.

board: "koth"

Set this to the board you want to automatically change to.

I had some issues with Featherboard and uppercase board names, so I recommend using lowercase boards.
range: 100

The range to how close the player has to be for the scoreboard to show up. Use -1 to enable it throughout the world.

rangemargin: 5

The error-margin of the range.

If you have the range on 100, then rangemargin 5 means you need to be:
- 100-5=95 blocks in range for the board to show up
- 100+5=105 blocks out of range for the board to get rid of it.

koth

remove-chest-at-start

Set this to false if you don't want the chest to be removed at the start

ffa-on-time-limit

If set to true, once the time is up and no capper won, it enters a Free-for-all state where it is a race to who gets the loot first.

default-capturetype

Set the default config type that will be set at restart/reload

contest-freeze

This config option makes it so all non-related players need to be off the point before you can start capping it (So all players that are not in the faction for example)

capture-cooldown

Once the player is off the point, the KoTH will enter a cooldown state, and takes X amount of seconds before it can be capped again.

previously called knock-time

channel-time

Require a player to stand on the koth for a configurable amount of seconds before actually capping it.

knock-time

This is the time how many seconds you can be off the KoTH before you actually leave it.

Previous knock-time was renamed to capture-cooldown

minimum-players

How many players at least need to be online for a KoTH to start

map-rotation

A list of KoTH's. You can then use "/koth start $rotation" to start from the top and move down the list.

This also works for scheduling
("/koth schedule create $rotation monday 5:00AM")
