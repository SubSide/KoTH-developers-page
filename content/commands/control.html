Koth editor commands

For these commands you need the permission: koth.admin.edit

The koth editor commands are:
  • start
  • end
  • stop
  • mode
  • change

/koth start <koth> [captureTime] [maxRunTime] [lootAmount] [entityType] Permission: koth.control.start Default: Operators

Starts a certain koth, you can run multiple koths at once! Just not the same koth multiple times.

Capturetime

The amount of time a player has to stand on the KoTH to capture it

Default: 15

maxRunTime

Maximum amount of time the KoTH can run

Default: -1 (unlimited)

lootAmount

How much loot should spawn

Default: (grabbed from the config)

entityType

With what entity type you want to start the koth
(For example: Player or Faction)

Default: (currently selected entity with /koth mode entitytype

/koth end [koth] Permission: koth.control.end Default: Operators

This will gracefully end all koths (or a specific koth with the argument). Gracefully means, that the person standing on it at that moment will instantly win it. If there is no person standing on it, it will be capped and won by the first person that will stand on it.

/koth stop [koth] Permission: koth.control.stop Default: Operators

This will forcefully stop all koths (or a specific koth with the argument). This means that it will just stop it like it never happened, no winners. Very helpful for testing purposes!

/koth mode [mode] Permission: koth.control.mode Default: Operators

/koth mode capturetype <capturetype>

Use this command to change the capturetype.

Current supported capturetypes:

  • player
  • factions (if Factions is enabled)
  • factionsuuid (if FactionsUUID is enabled)
  • kingdoms (if Kingdoms is enabled)
  • factions (if the FactionsOne extension is installed)

/koth mode gamemode <gamemode>

Current supported gamemodes:

  • classic
  • conquest (Only if a group-based entity is enabled)

Both capturetypes and gamemodes can be implemented by developers if they so desire.

/koth change Permission: koth.admin.change Default: Operators
