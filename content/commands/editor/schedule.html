Schedule commands

For these commands you need the permission: koth.admin.schedule

The schedule commands are:
  • create
  • edit
  • remove
  • list
  • asmember
  • clear
  • debug

/koth schedule create <koth> <day|(daily)> <time> [capturetime] [maxruntime] [lootamount] [lootchest]

Now this is a hell of a long command, and I know it. Luckily most of the arguments are not nessesary.

Let me explain a bit what the default values are for every command. And what they do.
This way I don't have to explain those down below in the edit section.

If it's easier, you can choose to only use the required arguments, and just edit it later with the edit command.

<koth>

This one should speak for itself, which Koth should run on this time.

<day>

The day of the week (monday, tuesday, wednesday, thursday, friday, saturday, sunday)

You can use a comma to make it easier to schedule multiple at once. (Don't use a space, so "Monday,Friday")

You can also use "daily", meaning it will automatically create 7 schedules for you, each day of the week.

<time>

The time the koth should take place.

You can use a comma to make it easier to schedule multiple at once. (Don't use a space, so "14:00,10AM")

Syntax is just 4:30PM or 21:50, or 5AM, just don't use spaces and you should be good. AM just doesn't do anything really, and PM just adds 12 to it (Except for 12, because you americans are that weird to not just add 12 to everything (jk jk))

[capturetime]

The time a player has to stand on the koth to capture it.

Default: 15

[maxruntime]

The maximum amount of minutes a koth can run. If it is being capped at the moment this timer runs out, it will wait till the player has capped it, or when he gets knocked off. If he gets knocked off when the timer already ran out, it will immediately end it.

Default: -1 (disabled)

[lootamount]

The amount of loot that should spawn in the chest.

Default: (The amount set in the config)

[loot]

Which lootchest should be used for this koth.

Default: (inherited by the KoTH itself)

/koth schedule edit

All schedule edit command require an ID,
the ID you can grab from /koth schedule list

I'm going to mention the commands here, I'm not going in-depth though.
All the meanings for each variable you can get above from: Here

  • /koth schedule edit <ID> koth <kothname>
  • /koth schedule edit <ID> day <day>
  • /koth schedule edit <ID> time <time>
  • /koth schedule edit <ID> capturetime <capturetime>
  • /koth schedule edit <ID> maxruntime <maxruntime>
  • /koth schedule edit <ID> lootamount <lootamount>
  • /koth schedule edit <ID> loot <loot>

/koth schedule remove <id>

Remove a certain schedule with the given #ID.

Please keep in mind, that the ID may differ every time you remove a schedule. So don't just blindly note all the ID's and start deleting them. You could cause a lot of problems!

/koth schedule list

Shows a list of all the created schedules with their IDs.

Please keep in mind, that the ID may differ every time you add/remove a schedule.

/koth schedule asmember

Members see the /koth schedule differently than an Administrator, members will see a nice marked up list. Now to make sure you know what members actually see you can use this command.

/koth schedule clear

Clears the whole schedule list

/koth schedule debug

A small debug command for testing purposes, /koth info schedule is way more recommended!
